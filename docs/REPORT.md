# 반도체 웨이퍼 맵 결함 패턴 자동 분류

딥러닝 기반 웨이퍼 결함 분류 모델 개발 프로젝트 — 전 과정 정리

---

## 1. 프로젝트 개요

### 1.1 배경

반도체 제조 공정에서 웨이퍼 위 수많은 다이(die)의 양·불량 여부를 검사하면, 불량 다이의 분포가 특정한 공간 패턴을 이룬다. 이 패턴(예: 가장자리에 몰린 불량, 중앙 집중, 긁힘 형태 등)은 공정상의 특정 이상 원인과 연결되기 때문에, 패턴을 정확히 분류하는 것은 수율 관리와 공정 진단의 출발점이 된다.

기존에는 엔지니어가 웨이퍼 맵을 눈으로 보고 패턴을 분류했으나, 물량이 많고 사람마다 판정이 달라지는 한계가 있다. 본 프로젝트는 이 분류 작업을 딥러닝 모델로 자동화하는 것을 목표로 한다.

### 1.2 목표

1. 웨이퍼 맵 이미지를 입력받아 결함 패턴을 9개 범주로 자동 분류하는 모델 구축
2. 클래스 불균형(정상 웨이퍼가 대다수)이 심한 실제 데이터에서 소수 결함 클래스까지 안정적으로 분류
3. 모델이 어떤 영역을 근거로 판단했는지 시각적으로 설명(설명가능 AI)
4. 분류된 결함 패턴에 대응하는 **공정 원인을 추론**하고 현장 **조치 가이드**를 제시
5. 정량 지표로 성능을 검증하고 개선 효과를 입증

### 1.3 성공 기준

정상(none)이 전체의 약 85%를 차지하므로 단순 정확도(accuracy)는 의미가 약하다. 따라서 **모든 클래스를 동등하게 평가하는 macro-F1**과 **클래스별 precision/recall**을 핵심 지표로 삼는다.

## 2. 데이터셋

### 2.1 원본 데이터

공개 벤치마크 데이터셋 **WM-811K**(파일명 `LSWMD.pkl`)를 사용했다. 실제 반도체 팹에서 수집된 웨이퍼 맵 모음으로, 결함 패턴 분류 연구의 표준 데이터셋이다.

| 항목 | 값 |
|---|---|
| 전체 웨이퍼 맵 수 | 811,457장 |
| 주요 컬럼 | `waferMap`, `dieSize`, `lotName`, `waferIndex`, `trianTestLabel`, `failureType` |
| 웨이퍼 맵 형식 | 2차원 격자, 칸 값 = 0(미검사/배경) · 1(정상 다이) · 2(불량 다이) |
| 맵 크기 | 가변 (웨이퍼마다 다름, 예: 45×48) |

### 2.2 라벨 체계 (9개 클래스)

`Center`, `Donut`, `Edge-Loc`, `Edge-Ring`, `Loc`, `Near-full`, `Random`, `Scratch`, `none`(결함 없음)

## 3. 데이터 전처리

원본 데이터를 모델 학습에 바로 쓸 수 없으므로, 라벨 정리 → 필터링 → 크기 통일 → 학습/검증/평가 분할의 단계를 거쳐 정제된 데이터셋을 구축했다.

### 3.1 라벨 정규화 및 탐색적 분석 (EDA)

원본의 `failureType` 필드는 문자열과 중첩 배열이 섞여 있어, 이를 일관된 문자열 라벨로 정규화했다. 그 결과 전체 81만 장 중 **라벨이 있는 샘플은 약 17.3만 장(약 21%)** 이고 나머지는 미라벨(unlabeled) 상태임을 확인했다.

라벨링된 샘플의 클래스 분포는 다음과 같다(크기 필터 적용 후).

| 클래스 | 샘플 수 | 비중 |
|---|---|---|
| none (정상) | 147,425 | 85.2% |
| Edge-Ring | 9,677 | 5.6% |
| Edge-Loc | 5,189 | 3.0% |
| Center | 4,292 | 2.5% |
| Loc | 3,588 | 2.1% |
| Scratch | 1,190 | 0.7% |
| Random | 865 | 0.5% |
| Donut | 555 | 0.3% |
| Near-full | 149 | 0.1% |

**핵심 발견**: 정상(none)이 85%를 차지하고, 가장 적은 Near-full은 149장에 불과한 극심한 클래스 불균형. 이 특성은 이후 모델링 전략(클래스 가중치, Focal Loss)을 결정짓는 근거가 되었다.

웨이퍼 크기 분포도 함께 분석하여, 비정상적으로 작거나 큰 웨이퍼(높이·너비 20~200 범위 밖)를 제외했다. 최종 학습 대상은 **172,930장**으로 확정하고, 매번 81만 장 전체를 다시 읽지 않도록 대상 인덱스를 `labeled_index.parquet`로 별도 저장했다.

### 3.2 이미지 변환 및 데이터셋 분할

크기가 제각각인 웨이퍼 맵을 모델 입력으로 쓰기 위해 다음을 수행했다.

1. **크기 통일**: 각 웨이퍼 맵을 128×128로 리사이즈. 0/1/2의 원래 값이 보간으로 흐려지지 않도록 **최근접 이웃 보간(nearest-neighbor)** 을 사용했다.
2. **배열화**: 전체를 `(172930, 128, 128)` uint8 배열(X)과 라벨 배열(y)로 변환 (약 2.8GB).
3. **분할**: 클래스 비율을 유지하는 **stratified split**으로 train / validation / test = 70% / 15% / 15% 분할.

| 분할 | 샘플 수 |
|---|---|
| train | 121,050 |
| validation | 25,940 |
| test | 25,940 |

분할 결과는 압축 형식(`wafer_train.npz`, `wafer_val.npz`, `wafer_test.npz`)으로 저장하여, 이후 모델 실험 시 전처리를 반복하지 않고 즉시 로드할 수 있게 했다.

## 4. 방법론

### 4.1 모델 아키텍처 — 전이학습

이미지 분류에 검증된 **ResNet18** 합성곱 신경망을 사용했다. 바닥부터 학습하는 대신, 대규모 일반 이미지(ImageNet)로 **사전학습된 가중치**를 출발점으로 삼아 마지막 분류층만 9개 클래스에 맞게 교체하고 웨이퍼 데이터로 재학습(파인튜닝)했다. 사전학습 모델이 이미 익힌 선·모서리·질감 인식 능력을 활용하므로, 적은 데이터로도 빠르고 안정적으로 수렴한다.

입력 처리: 웨이퍼 맵(0/1/2)을 0~1로 정규화한 뒤 약 [-1, 1] 범위로 표준화하고, ResNet 입력 규격에 맞춰 3채널로 복제했다. 웨이퍼 맵은 회전·반전에 대해 결함 종류가 바뀌지 않으므로, 학습 시 90도 단위 회전과 좌우·상하 반전을 데이터 증강으로 적용했다.

공통 학습 설정: 옵티마이저 AdamW(학습률 3e-4, weight decay 1e-4), 코사인 스케줄러, 6 에폭, 검증 macro-F1이 가장 높은 시점의 가중치를 저장.

### 4.2 클래스 불균형 대응 — 2단계 접근

**1차 (기준 모델)**: 클래스 빈도의 역수(inverse-frequency)를 손실 가중치로 사용했다. 희귀 클래스 오류에 큰 페널티를 주는 방식이다.

**2차 (개선 모델)**: 1차 모델 분석에서 가중치가 지나치게 공격적(최대 20배)이라 모델이 결함을 과도하게 예측 → 오탐(false positive)이 많다는 문제를 발견했다. 이를 해결하기 위해 두 가지를 도입했다.

- **Class-balanced 가중치(effective number of samples)**: 빈도 역수보다 훨씬 완만한 가중치(최대 약 4배)로 조정하여 결함 남발을 억제.
- **Focal Loss (γ=2)**: 이미 쉽게 맞히는 샘플(주로 정상)의 기여를 줄이고, 헷갈리는 샘플에 학습을 집중.

### 4.3 설명가능성 — Grad-CAM

모델이 웨이퍼의 어느 영역을 근거로 판단했는지 확인하기 위해 **Grad-CAM**을 적용했다. 마지막 합성곱 블록(layer4)의 활성화와 기울기를 결합해 관심 영역을 히트맵으로 시각화하는 기법으로, 모델이 실제 결함 위치를 보고 판단하는지 검증하는 데 사용했다.

### 4.4 공정 원인 추론 — 멀티태스크 학습

결함 종류 분류에서 한 걸음 더 나아가, 결함을 유발한 **공정 원인**까지 추론하도록 모델을 확장했다. ResNet18 백본을 공유하고 그 위에 두 개의 분류 헤드를 두는 **멀티태스크 구조**다.

- `defect_head`: 결함 종류 9개 분류 (Focal Loss + class-balanced 가중치)
- `cause_head`: 공정 원인 9개 분류 (lithography_cmp, pr_uniformity, particle_edge, particle_handling, handling, equipment, environment, mixed, normal)

원인 라벨은 제안서 2.1절의 결함→원인 대응표(`configs/mappings/defect_to_cause.yaml`)에서 생성했다. 학습은 두 태스크의 손실을 합산(`결함손실 + 0.5 × 원인손실`)하여 동시에 수행했다. 추론 시에는 결함 종류와 공정 원인을 함께 출력하고, 원인별 **조치 가이드**(예: 장비 점검 항목)까지 제시하여 분류 결과를 곧바로 현장 대응으로 연결한다.

## 5. 실험 결과

### 5.1 기준 모델 (1차)

| 지표 | 값 |
|---|---|
| accuracy | 0.943 |
| **macro-F1** | **0.811** |

클래스별로 recall은 전반적으로 높았으나(평균 0.94) precision이 낮은 클래스가 존재했다. 특히 Scratch(precision 0.42), Loc(0.56), Edge-Loc(0.60)은 헛알람이 많았다. 이는 공격적 가중치로 인해 모델이 결함을 과잉 예측한 결과로 분석되었다.

### 5.2 개선 모델 (2차)

| 지표 | 1차 | 2차 | 변화 |
|---|---|---|---|
| accuracy | 0.943 | 0.981 | +0.038 |
| **macro-F1** | **0.811** | **0.899** | **+0.088** |

클래스별 precision 개선(오탐 감소):

| 클래스 | precision(1차) | precision(2차) | 변화 |
|---|---|---|---|
| Center | 0.726 | 0.951 | +0.225 |
| Donut | 0.811 | 0.854 | +0.043 |
| Edge-Loc | 0.596 | 0.877 | +0.281 |
| Edge-Ring | 0.957 | 0.990 | +0.033 |
| Loc | 0.558 | 0.867 | +0.309 |
| Near-full | 0.815 | 0.846 | +0.031 |
| Random | 0.688 | 0.884 | +0.196 |
| Scratch | 0.424 | 0.863 | +0.439 |
| none | 0.998 | 0.989 | −0.009 |

가장 약했던 Scratch의 precision이 0.42에서 0.86으로 약 2배 개선되는 등, 모든 결함 클래스에서 오탐이 크게 줄었다. 정상(none)의 precision은 거의 유지되어, 정상품을 결함으로 오판하는 비율도 늘지 않았다. 일부 클래스(Loc, Scratch, Edge-Loc)는 recall이 소폭 감소했는데, 이는 "헛알람을 줄이는 대신 일부 결함을 놓칠 수 있는" precision-recall 트레이드오프로, 전체 균형(macro-F1)은 크게 향상되었다.

### 5.3 Grad-CAM 분석

개선 모델의 관심 영역을 시각화한 결과, 모델이 배경이나 무관한 영역이 아니라 **실제 결함 패턴이 위치한 영역**에 주목하고 있음을 확인했다. 예를 들어 Edge-Ring은 웨이퍼 가장자리 링 영역, Edge-Loc은 가장자리의 국소 영역에 활성화가 집중되었다. 이는 모델이 우연이 아니라 의미 있는 시각적 근거로 판단하고 있음을 뒷받침한다.

### 5.4 멀티태스크 모델 — 결함 + 공정 원인

백본을 공유하는 멀티태스크 모델로 결함 종류와 공정 원인을 동시에 예측했다. 장기 학습(50에폭 + 조기종료)과 TTA를 적용한 최종 멀티태스크 모델의 성능은 다음과 같다.

| 태스크 | macro-F1 | accuracy |
|---|---|---|
| 결함 종류 (9-class) | 0.906 | 0.981 |
| 공정 원인 (9-class) | 0.904 | 0.981 |

원인 헤드는 결함 헤드와 유사한 수준으로 함께 향상되었으며(각 결함이 단일 주요 원인에 1:1 대응하므로 원인 성능은 결함 성능을 따라간다), 두 태스크를 함께 학습해도 결함 분류 성능 저하 없이 원인 추론 기능을 추가할 수 있음을 확인했다. 추론 데모에서는 임의의 웨이퍼 맵에 대해 결함 종류·공정 원인·조치 가이드가 정상적으로 출력되었다.

예시 출력:

```
[실제 Edge-Ring] 결함예측=Edge-Ring(100%)  원인=particle_edge(100%)
     조치: 베벨 클리너 점검, CMP 슬러리 공급부 점검, EBR 노즐 상태
[실제 Center]    결함예측=Center(99%)       원인=lithography_cmp(100%)
     조치: 노광 포커스/도즈 점검, CMP 패드 컨디셔닝 상태 확인
```

### 5.5 장기 학습 + TTA — 최종 결함 분류 모델

개선 모델(2차)의 학습을 6에폭에서 **50에폭(조기종료)** 으로 늘리고, 추론 시 회전·반전한 8가지 버전의 예측을 평균하는 **TTA(test-time augmentation)** 를 적용해 성능을 추가로 끌어올렸다.

| 구성 | macro-F1 | accuracy |
|---|---|
